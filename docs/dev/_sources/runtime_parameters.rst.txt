integration
===========

+---------------------------------------+---------------------------------------------------------+--------------------+
| parameter                             | description                                             | default value      |
+=======================================+=========================================================+====================+
| ``do_constant_volume_burn``           | When evolving the temperature, should we assume a       | .true.             |
|                                       | constant volume (default) or a constant pressure        |                    |
|                                       | (do_constant_volume_burn = F)?                          |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``call_eos_in_rhs``                   | Normally we update the temperature during a burn to be  | .true.             |
|                                       | consistent with the current internal energy. This is    |                    |
|                                       | done with an EOS call, which can be turned off if       |                    |
|                                       | desired. This will freeze the temperature and specific  |                    |
|                                       | heat to the values at the beginning of the burn, which  |                    |
|                                       | is inaccurate but cheaper.                              |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``dT_crit``                           | If we want to call the EOS in general, but don't want   | 1.0d20             |
|                                       | to overdo it, we can set a fraction ``dT_crit`` such    |                    |
|                                       | that we only do the EOS call if the temperature has     |                    |
|                                       | changed by a relative fraction > ``dT_crit``. If we use |                    |
|                                       | this option, we will do a linear fit to c_v and c_p in  |                    |
|                                       | between EOS calls. This will work regardless of         |                    |
|                                       | call_eos_in_rhs.                                        |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``integrate_energy``                  | Allow the energy integration to be disabled by setting  | .true.             |
|                                       | the RHS to zero.                                        |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``jacobian``                          | Whether to use an analytical or numerical Jacobian. 1   | 1                  |
|                                       | == Analytical 2 == Numerical 3 == Numerical for         |                    |
|                                       | reactions; analytic for primitive -> conserved          |                    |
|                                       | (simplified-SDC only)                                   |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``burner_verbose``                    | Should we print out diagnostic output after the solve?  | .false.            |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``rtol_spec``                         | Tolerances for the solver (relative and absolute), for  | 1.d-12             |
|                                       | the species and energy equations.                       |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``rtol_enuc``                         |                                                         | 1.d-6              |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``atol_spec``                         |                                                         | 1.d-8              |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``atol_enuc``                         |                                                         | 1.d-6              |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``retry_burn``                        | If we fail to find a solution consistent with the       | .false.            |
|                                       | tolerances, do we want to try again with a looser       |                    |
|                                       | tolerance?                                              |                    |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``abort_on_failure``                  | Should we abort the run when the burn fails?            | .true.             |
+---------------------------------------+---------------------------------------------------------+--------------------+
| ``renormalize_abundances``            | Whether to renormalize the mass fractions at each step  | .false.            |

| ``SMALL_X_SAFE``                      | The absolute cutoff for species -- note that this might | 1.0d-30            |
