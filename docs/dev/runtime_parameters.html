integrationÂ¶

parameter

description

default value

do_constant_volume_burn

When evolving the temperature, should we assume a constant volume (default) or a constant pressure (do_constant_volume_burn = F)?

.true.

call_eos_in_rhs

Normally we update the temperature during a burn to be consistent with the current internal energy. This is done with an EOS call, which can be turned off if desired. This will freeze the temperature and specific heat to the values at the beginning of the burn, which is inaccurate but cheaper.

.true.

dT_crit

If we want to call the EOS in general, but donât want to overdo it, we can set a fraction dT_crit such that we only do the EOS call if the temperature has changed by a relative fraction > dT_crit. If we use this option, we will do a linear fit to c_v and c_p in between EOS calls. This will work regardless of call_eos_in_rhs.

1.0d20

integrate_energy

Allow the energy integration to be disabled by setting the RHS to zero.

.true.

jacobian

Whether to use an analytical or numerical Jacobian. 1 == Analytical 2 == Numerical 3 == Numerical for reactions; analytic for primitive -> conserved (simplified-SDC only)

1

centered_diff_jac

one-sided numerical jacobian (.False.) or centered- difference Jacobian (.true.). Note: the centered- difference requires twice as many RHS calls

.false.
